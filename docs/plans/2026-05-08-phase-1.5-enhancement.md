## L1 评分标准

本节为 Issue 2（分层内容）中使用的详细评分标准。

### 9-10 分：必读

- 重大 AI/ML 突破性公告：新模型发布（GPT、Claude、Gemini、DeepSeek、Qwen 等）
- 核心软件工程范式转变（新编程模型、架构革命）
- AI/ML 工具链中的重要开源项目发布（LangChain、PyTorch、transformers、vLLM、llama.cpp）
- 具有独特见解的权威性技术深度分析
- 对 AI/ML 开发有直接实践意义的研究论文
- 与 AI/ML 从业者日常工作直接相关——错过会后悔

### 7-8 分：高度相关

- 带有可运行代码示例的实用 AI/ML 教程
- 开发者工具评测、对比、基准测试
- LLM 集成指南、提示工程技巧、RAG 模式
- 影响开发者工具的重大安全漏洞或 CVE
- 有量化结果的性能优化深度分析
- 新编程语言特性或重大框架更新
- 一周内可直接应用——能用到当前工作中

### 5-6 分：中等相关

- 科技行业新闻：融资、收购、产品发布
- 创业故事和创始人访谈
- 通用软件工程文章（非 AI 专项，如 Kubernetes、数据库内核）
- 影响行业的科技政策、法规或法律分析
- 职业成长、工程管理、效率建议
- 有趣的背景信息但不可立即应用

### 3-4 分：低相关

- 消费级软件/应用评测（手机 App、游戏、社交媒体）
- 没有技术/工程视角的通用科学文章（生物、物理、天文）
- 科技周边文化内容（硅谷剧集评论、科技名人八卦）
- 面向非工程师的营销/增长黑客文章
- 硬件/设备评测（手机、笔记本、IoT）与软件工程无关
- 对 AI/ML 从业者无可操作性

### 1-2 分：不相关

- 娱乐、生活、体育、旅游
- 与科技无关的历史散文、个人叙事
- 幽默、讽刺、梗图内容
- 政治评论、非科技话题的观点文章
- 完全跑题——与技术或工程工作毫无关联

## Issue 4：YAML 配置文件管理 RSS 源

**目标：** 用 YAML 配置文件替代 `config.py` 中硬编码的 RSS 源。用户可在 `pipeline/configs/*.yaml` 维护多套配置方案，通过 `--config` 命令行参数选择。

**YAML 配置格式：**

```yaml
# pipeline/configs/tech-feeds.yaml
name: tech-feeds
sources:
  - name: 品玩
    url: https://plink.anyfeeder.com/appinn
    category: news
    max_articles: 10
  - name: 少数派
    url: https://sspai.com/feed
    category: tech
    max_articles: 5
```

**命令行行为：**
- `python -m pipeline.main --source rss` → 加载 `pipeline/configs/default.yaml`
- `python -m pipeline.main --source rss --config pipeline/configs/ai-feeds.yaml` → 加载指定文件
- `python -m pipeline.main --source rss --config my-custom.yaml` → 相对于 `pipeline/configs/` 解析路径

## Issue 1：按日期分组文章目录

**目标：** 文章按日期分组存放：`vault/2-Articles/YYYY-MM-DD/article-slug.md`，替代扁平的 `vault/2-Articles/article-slug.md`。

**日期来源：** 使用 `date_processed`（管线运行日期 = `datetime.now().strftime("%Y-%m-%d")`），因为很多 RSS 文章缺少 `published_at` 字段。

**涉及文件：**
- `pipeline/formatters/obsidian_writer.py` — `_get_target_dir()` 追加日期子目录
- `pipeline/tests/test_obsidian_writer.py` — 更新目录断言

## Issue 2：评分分层内容深度

**目标：** 基于 L1 的 relevance_score 将内容分为三个处理层级。这是最复杂的改动——涉及 L1 提示词、L2 处理逻辑、Writer 渲染和主管线流程。

**层级定义：**

| 分数范围 | 层级 | L2 行为 | .md 中的内容 |
|----------|------|---------|-------------|
| 1-3 | **丢弃** | 完全跳过 | 不写入 vault |
| 4-6 | **压缩** | 当前 L2：摘要 + 关键要点（高度压缩） | 摘要 + 关键要点 |
| 7-10 | **详细** | L2 + 原文轻度压缩（保留原文） | 摘要 + 关键要点 + 原文内容 |

**配置变更：** 用分层边界配置替代单一的 `relevance_threshold`。

---

## Issue 3：文件名嵌入重要性评分

**目标：** 在文件名中嵌入评分，用户无需打开文件即可在 Obsidian 文件浏览器中扫描文章重要性。

**设计方案：** 使用 `[S8]` 前缀格式：`[S8] article-slug.md`。紧凑且在 Obsidian 文件浏览器中视觉可扫描。
