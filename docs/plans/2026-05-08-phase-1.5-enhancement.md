## L1 Scoring Rubric (User Request: "文章分值判断规则是怎样需要给我一个较详细的标准")

This is the detailed scoring standard to be used in Issue 2 (tiered content). Presenting it first as the user asked for it explicitly.

### Score 9-10: 必读 (Must Read)

- Major AI/ML breakthrough announcements: new model releases (GPT, Claude, Gemini, DeepSeek, Qwen, etc.)
- Core software engineering paradigm shifts (new programming models, architecture revolutions)
- Significant open-source releases in AI/ML toolchain (LangChain, PyTorch, transformers, vLLM, llama.cpp)
- Authoritative technical deep-dives with novel insights not found elsewhere
- Research papers with immediate practical implications for AI/ML development
- Directly relevant to daily AI/ML practitioner work — would regret missing

### Score 7-8: 高度相关 (Highly Relevant)

- Practical AI/ML tutorials with working code examples
- Developer tool evaluations, comparisons, benchmarks
- LLM integration guides, prompt engineering techniques, RAG patterns
- Significant security vulnerabilities or CVEs affecting developer tools
- Performance optimization deep-dives with measurable results
- New programming language features or major framework updates
- Directly actionable within a week — can apply to current work

### Score 5-6: 中等相关 (Moderately Relevant)

- Tech industry news: funding rounds, acquisitions, product launches
- Startup stories and founder interviews
- General software engineering articles (not AI-specific, e.g., Kubernetes, database internals)
- Tech policy, regulation, or legal analysis affecting the industry
- Career growth, engineering management, productivity advice
- Interesting context but not immediately actionable

### Score 3-4: 低相关 (Low Relevance)

- Consumer software/app reviews (phone apps, gaming, social media)
- General science articles (biology, physics, astronomy) without a tech/engineering angle
- Tech-adjacent culture content (Silicon Valley TV show reviews, tech celebrity gossip)
- Marketing/growth hacking articles aimed at non-engineers
- Hardware/gadget reviews (phones, laptops, IoT) without software engineering relevance
- Not actionable for AI/ML practitioner

### Score 1-2: 不相关 (Irrelevant)

- Entertainment, lifestyle, sports, travel
- Historical essays, personal narratives without tech connection
- Humor, satire, meme content
- Political commentary, opinion pieces on non-tech topics
- Completely off-topic — no connection to technology or engineering work

def load_rss_config(config_path: Optional[str] = None) -> List[RSSSource]:
    """Load RSS sources from a YAML config file.

    Args:
        config_path: Path to YAML file. If None, tries 'pipeline/configs/default.yaml'.
                     If the default doesn't exist, falls back to hardcoded defaults.

    Returns:
        List of RSSSource objects.
    """
    if config_path is None:
        config_path = str(PIPELINE_DIR / 'configs' / 'default.yaml')

    path = Path(config_path)
    if not path.is_absolute():
        path = PIPELINE_DIR / 'configs' / path.name

    if not path.exists():
        import logging
        logger = logging.getLogger(__name__)
        logger.warning(f"Config file not found: {path}, using hardcoded defaults")
        return _default_rss_sources()

    with open(path, 'r', encoding='utf-8') as f:
        data = yaml.safe_load(f)

    sources = []
    for item in data.get('sources', []):
        sources.append(RSSSource(
            name=item['name'],
            url=item['url'],
            category=item.get('category', 'general'),
            max_articles=item.get('max_articles', 20),
            enabled=item.get('enabled', True),
        ))

    return sources

### Task 2.7: Make L1 filter_batch tier-aware

**Objective:** `filter_batch()` assigns tiers and filters out "discard" tier.

**Files:**
- Modify: `pipeline/processors/l1_filter.py`

**Step 1: Add `_assign_tier()` method to L1Filter class**

```python
def _assign_tier(self, article: Article, tier_discard_max: int, tier_compressed_max: int) -> str:
    """Assign content tier based on relevance score."""
    score = article.relevance_score
    if score <= tier_discard_max:
        return "discard"
    elif score <= tier_compressed_max:
        return "compressed"
    else:
        return "detailed"
```

**Step 2: Update `filter_batch()` signature and logic**

Change signature from:
```python
def filter_batch(self, articles: List[Article], threshold: int = 6) -> List[Article]:
```
To:
```python
def filter_batch(self, articles: List[Article], config) -> List[Article]:
```

Update body:
```python
def filter_batch(self, articles: List[Article], config) -> List[Article]:
    """Filter a batch of articles. Assigns tiers, returns only compressed+detailed."""
    filtered = []
    discarded = 0
    for article in articles:
        scored = self.filter_article(article)
        if scored and scored.relevance_score > 0:
            scored.content_tier = self._assign_tier(
                scored, config.tier_discard_max, config.tier_compressed_max
            )
            if scored.content_tier == "discard":
                discarded += 1
                logger.info(
                    f"L1 discarded (score={scored.relevance_score}): "
                    f"'{article.title[:40]}...'"
                )
            else:
                filtered.append(scored)
        else:
            discarded += 1
    logger.info(
        f"L1 batch: {len(articles)} in -> {len(filtered)} passed, {discarded} discarded "
        f"(discard<={config.tier_discard_max}, compressed<={config.tier_compressed_max})"
    )
    return filtered
```

**Step 3: Update `filter_article()` return**

Change return type from `Optional[Article]` to `Article` (never returns None now — always returns the article with score). Update docstring.
