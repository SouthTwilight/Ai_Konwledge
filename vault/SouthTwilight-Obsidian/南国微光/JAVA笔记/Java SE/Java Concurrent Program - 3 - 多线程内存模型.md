### Java 内存模型

Java 内存模型 JMM，即 Java Memory Model 。定义了 Java 语言中主存、工作内存的抽象概念，底层对应着 CPU 寄存器、缓存、硬件内存、CPU 指令优化等

**特点：**

- _**原子性**_ - 保证**指令不会受到线程上下文切换的影响**
- _**可见性**_ - 保证**指令不会受 CPU 缓存的影响**
- _**有序性**_ - 保证**指令不会受 CPU 指令并行优化的影响**

### 可见性

> 可见性即 **“内存可见性”** ，**共享变量**才能谈其可见性。共享变量被多个线程使用时，是各个线程将公共位置的值拷贝一份本线程私有空间中使用。当某个线程对共享变量进行了修改，并写入到共享变量的公共位置时，其他线程中共享变量值未跟随一起被改变。可见性就是指：**当共享变量被某个线程中改写时，其他线程在已经读取该变量值但还未使用该共享变量前能检测到该变化并获取变化后的共享变量值。**

 ##### **保证共享变量对多线程的可见性：**
- **volatile 关键字修饰需要被共享的变量，volatile(** 不稳定的 **)可以避免线程直接使用自身高速缓存中的共享变量值，使线程在使用共享变量时必须去主存中共享变量的公共位置处检查共享变量的值是否被改变。线程使用被 volatile 修饰的变量时将直接操作主存。**
- **可见性**与**原子性**的关系与区别
    1. 可见性保证在多个线程访问同个共享变量时，**其中一个线程对 volatile 修饰的变量修改对另外的线程可见， 但不能保证共享变量的_原子性_。**可见性仅适用于一个写线程，多个读线程。
    2. 原子性是指一个线程的操作是不能被其他线程打断，同一时间只有一个线程对一个变量进行操作。而若有**多个线程对同一个共享变量进行写操作时，由于多线程存在指令交错的情况，可能会导致最终结果与预期结果不符**。即使满足了可见性，但依旧不能避免多个写线程的指令交错情况。
    3. synchronized 语句块可以**保证代码块的原子性**与**代码块内变量的可见性**。但缺点是 synchronized 是属于重量级操作，性能相对较低

 ##### _**内存屏障：**_
- 定义
    1. 写屏障：保证在该写屏障之前，所有对该写屏障所属共享变量的改动都被同步到主存
    2. 读屏障：保证在该读屏障之后，所有线程对该读屏障所属共享变量的读取都是获取的主存中最新数据
- 作用
    1. 写屏障会确保指令重排序时，不会将写屏障之前的代码排在写屏障之后
    2. 读屏障会确保指令重排序时，不会将读屏障之后的代码排在读屏障之前

#📌 **System.out.println()** 方法上被加上了 synchronized 关键字，保证**打印操作是一个原子操作**


### 线程同步模式—— _**Balking ( 犹豫 )模式**_

> **定义：**一个线程发现另外线程或本线程**正在执行_已完成的_** 某个**只需执行一次或只有状态被改变时才继续执行的操作**时，本线程将停止执行相同操作而直接结束并返回。本质是**停止并返回。**

可以用于实现线程安全单例

```java
public final class Singleton {
		private Singleton() {}

		private static Singleton INSTANCE = null;

		public static synchronized Singleton getInstance() {
				if (INSTANCE != null) {
						return INSTANCE;
				}
				INSTANCE = new Singleton();
				return INSTANCE;
		}
}
```

### Java 程序的有序性

JVM 虚拟机在不影响程序执行的情况下，会对程序的指令执行进行优化重排，以提升程序执行效率。但在多线程执行状态下，**指令重排**机制会受到不同线程处理的影响，导致程序的指令经过重排后不能得到预期结果。

**指令执行步骤：**

- 通常情况下，一条指令的执行过程由 _**取指令 - 指令译码 - 执行指令 - 内存访问 - 数据写回**_ 等阶段组成

### _volatile 关键字_

> **JVM 赋予** volatile 特性： 
> 1. volatile 修饰变量，保证此变量对所有线程的可见性，改变后的新值将被所有使用该变量的线程得知。普通变量值的传递通过主存完成，通常对于不同线程具有不同的得到新值速度。 
> 2. 禁止指令重排序优化。

#📌 _**volatile**_ **底层通过内存屏障实现：**
		1. **对 volatile **修饰的**变量写指令  _后_  **加入**写屏障，防止写屏障前的代码重排到对该变量修改之后**
		2. **对 volatile **修饰的**变量读指令  *前*  **加入**读屏障，防止读屏障后的代码重排到对该变量修改之前**
