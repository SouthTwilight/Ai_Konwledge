抽象类定义： 只需要在class前添加abstract关键字，无需其他访问权限控制符。

***注：***_抽象类无法实例化，无法创建对象。抽象类是用来被子类继承的。_

**特点：**

> 1.抽象类无法实例化，无法创建对象。抽象类是用来被子类继承的。 
> 2.final和abstract不能联合使用，两个关键字对立。 
> 3.抽象类的子类可以是抽象类和非抽象类。
> 4.抽象类无法实例化，但抽象类有构造方法，构造方法供子类使用。 
> 5.抽象类不一定有抽象方法，抽象方法必须出现在抽象类中。

抽象方法的定义：

👉 _public abstract return_type name();_

- _**注：一个非抽象的类，继承抽象类，必须将抽象类中的抽象方法进行覆盖/重写/实现。**_

**易错点：**

> **问：** java语言中凡是没有方法体的方法都是抽象方法?
> **答：** 不对。如Object类中有很多方法都没有方法体，以“;”结尾，但他们都不是抽象方法，
> 	例： _public native int hashCode();_ 
> 这个方法底层调用了C++写的动态链接库程序。前面修饰符列表中没有 'abstract修饰。有一个 ' native ' 。表示调用JVM本地程序。
