**接口基础概念：**

> 1、接口是“引用数据类型” 
> 2、接口完全抽象，即只能有常量与抽象方法 
> 3、接口定义：[ 修饰符列表 ] interface 接口名 { 接口内容 } 
> 4、接口支持多继承（即，一个接口可以继承多个接口 [ extends ]，一个类可以实现多个接口[implements]）
> 5、接口中元素和方法的访问控制权限都是public级别 6、接口中抽象方法前缀"public abstract"可省略，常量前缀"public static final"可省略
> 7、接口被非抽象类实现时，非抽象类必须将接口中所有方法一起实现。 
> 8、extends和implements可以共存：***public class 类名1 extends 类名2 implements 接口名***
> 9、使用接口时，可使用多态（父类型引用指向子类型对象）："接口类型 接口变量名 = new 非抽象类构造"

**接口的作用：**

> 接口的作用，类似多态的作用。即，面向抽象编程，不要面向具体编程。降低程序的耦合度。提高程序的扩展力。
> 
> 面向接口编程，可以降低程序的耦合度，提高程序的扩展力。符合OCP开发原则。接口的使用离不开多态机制。
> 
> 接口联合多态使用可以起到降低耦合度的作用。

***注：*_解耦合，将调用者（类）和实现者（类）解耦合，即降低类与类之间的关联程度_

**接口与抽象类：**

> 1.抽象类是半抽象的（允许具有非抽象方法），接口是完全抽象的（只能由抽象方法）
> 2.抽象类中有构造方法（默认无参数构造方法），接口中没有构造方法。
> 3.类和类之间只能单继承，类和接口之间支持多实现，接口和接口之间支持多继承。
