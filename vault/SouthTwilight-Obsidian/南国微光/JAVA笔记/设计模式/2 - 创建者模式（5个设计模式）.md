#### 一、工厂方法模式

>**定义：** 其在父类中提供一个创建对象的方法， 允许子类决定实例化对象的类型 
>**意图：** 定义一个创建对象的接口，由子类决定实例化工厂，使其创建过程延迟到子类进行 
>**优点：**     1、避免创建者与具体的产品逻辑耦合；
>	    2、满足单一职责，每一个业务逻辑实现都在所属自己的类中完成；
>	    3、满足开闭原则，无需更改使用调用方就可以在程序中引入新的产品类型

- 通常 Java 代码结构

```java
demo
└── src
    └── main
        └── java
            └── com.niexl.design
                ├── store    
                │   ├── impl       // 实际使用的类，实现对外接口及业务逻辑
                │   │   ├── CouponCommodityService.java 
                │   │   └── GoodsCommodityService.java  
                │   └── ICommodity.java  // 对外提供的统一接口
                └── StoreFactory.java    // 工厂类，具体负责创建对象
```
- 过程逻辑
	1. 工厂类实现（根据业务需求不同创建的对象类）
		```java
		public class StoreFactory {
		    public ICommodity getCommodityService(Integer commodityType) {
		        if (null == commodityType) return null;
		        if (1 == commodityType) return new CouponCommodityService();
		        if (2 == commodityType) return new GoodsCommodityService();
		        throw new RuntimeException("不存在的商品服务类型");
		    }
		}
		```
	2. 使用时传入不同条件，获取所需对象（sendCommodity 是接口对外提供统一业务接口）
		```java
		// 1. 优惠券
		ICommodity commodityService_1 = storeFactory.getCommodityService(1);
	    commodityService_1.sendCommodity("10001", "EGM1023938910232121323432", "791098764902132", null);
    
		// 2. 实物商品
	    ICommodity commodityService_2 = storeFactory.getCommodityService(2);
	    Map<String,String> extMap = new HashMap<String,String>();
	    extMap.put("consigneeUserName", "谢飞机");
	    extMap.put("consigneeUserPhone", "15200292123");
	    extMap.put("consigneeUserAddress", "吉林省.长春市.双阳区.XX街道.檀溪苑小区.#18-2109");
	    commodityService_2.sendCommodity("10001","9820198721311","1023000020112221113", extMap);
		```

#### 二、抽象工厂方法
> **定义：** *是一种**为访问类提供一个创建一组相关或相互依赖对象的接口，且访问类无须指定所要产品的具体类就能得到同族的不同等级的产品**的模式结构*
> **意图：** ***处理多等级产品（产品族）生产，某具体工厂生产位于不同等级的一组产品是一个产品族***
> **特点：** 生产多个等级的产品

- **通常 Java 代码结构**

```java
demo
└── src
    └── main
        └── java
            └── org.niexl.design
                ├── factory    
                │   ├── impl // 一个产品族
                │   │   ├── EGMCacheAdapter.java  // 真实产品类 EGM 的对外适配器实现,统一提供的函数名
                │   │   └── IIRCacheAdapter.java  // 真实产品类 IIR 的对外适配器实现
                │   ├── ICacheAdapter.java      // 函数适配器接口类，对外提供统一函数接口
                │   ├── JDKInvocationHandler.java // 代理类调用处理接口类，重写 invoke 方法
                │   └── JDKProxy.java // 代理类，代理生成不同真实产品实例的代理类
                ├── impl
                │   └── CacheServiceImpl.java // 产品实现类
                └── CacheService.java // 抽象产品的服务接口，对外提供服务（会被真实产品类服务覆盖）
```
- 1、JDKInvocationHandler (大致写法)
    ```java
    public class JDKInvocationHandler implements InvocationHandler {
        private ICacheAdapter cacheAdapter;
        public JDKInvocationHandler(ICacheAdapter cacheAdapter) {
            this.cacheAdapter = cacheAdapter;
        }
        public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
            return ICacheAdapter.class.getMethod(method.getName(), ClassLoaderUtils.getClazzByArgs(args)).invoke(cacheAdapter, args);
        }
    }
    ``` ^7b5f1a
- 2、JDKProxy （大致写法）
    ```java
    public class JDKProxy {
        public static <T> T getProxy(Class<T> interfaceClass, ICacheAdapter cacheAdapter) throws Exception {
            InvocationHandler handler = new JDKInvocationHandler(cacheAdapter);
            ClassLoader classLoader = Thread.currentThread().getContextClassLoader();
            Class<?>[] classes = interfaceClass.getInterfaces();
    		// 1、加载代理对象的类加载器;2、需要实现的接口;/3、动态代理方法执行时调用 handler 里面的invoke方法执行
            return (T) Proxy.newProxyInstance(classLoader, new Class[]{classes[0]}, handler);
        }
    }
    ```
 ^2e6139
- 常用方式
    ```java
    // 创建代理类
    CacheService proxy_EGM = JDKProxy.getProxy(CacheServiceImpl.class, new EGMCacheAdapter());
    // 调用统一的接口提供的方法
    proxy_EGM.set("user_name_01", "小傅哥");
    ```

#### 三、建造者模式
> **定义：** 将一个复杂对象的构造与它的表示分离，使同样的构建过程可以创建不同的表示的设计模式 
> **意图：** 将多个简单对象通过一步步的组装构建出一个复杂对象的过程 
> **特点：** 建造者模式注重零部件的组装过程，工厂方法模式更注重零部件的创建过程
- **通常 Java 代码结构**
```java
demo
└── src
    └── main
        └── java
            └── org.niexl.design
                ├── Builder.java    // 建造者类，负责实现并返回业务所需复杂对象
                ├── DecorationPackageMenu.java //业务方法实现类，负责实现接口提供的组件添加方法
                └── IMenu.java // 包装接口类，提供负责组建复杂对象的接口方法
```

- 1、IMenu（大致写法）
    ```java
    // 主要用于定义进行建造复杂对象的添加组件的方法
    public interface IMenu {
        IMenu appendCeiling(Matter matter);
        IMenu appendCoat(Matter matter);
        IMenu appendFloor(Matter matter);
        IMenu appendTile(Matter matter);
        String getDetail();
    }
    ```
- 2、DecorationPackageMenu（大致写法）
    ```java
    public class DecorationPackageMenu implements IMenu {
    	......
    	// 实现建造接口某个组装方法
		public IMenu appendCoat(Matter matter) {
			list.add(matter);
			// 通过某个算法为复杂的建造对象某个属性进行赋值
	        price = price.add(area.multiply(new BigDecimal("1.4")).multiply(matter.price()));
    		// 返回该复杂建造对象
	        return this;
		}
	      ......
    }
    ```
    
- 3、Builder（大致写法）

#### 四、原型**模式**

> **定义：** 用一个已经创建的实例作为原型，通过复制该原型对象来创建一个和原型**相同**或**相似**的新对象 
> **意图：** 消除**系统中需要大量相同或相似对象**的创建导致的复杂且耗时耗资源的问题 
> **特点：** 性能上优良；简化创建对象的过程；可辅助实现撤销操作

```java
demo
└── src
    └── main
        └── java
            └── org.niexl.design
                ├── util
                │   ├── Topic.java     // 进行目标对象不同于原型的处理类
                │   └── TopicRandomUtil.java // 处理类的工具类，视情况而定（非必要）
                ├── QuestionBank.java    // 原型基类，提供可使用的 clone 方法，用于对原型进行克隆
                └── QuestionBankController.java  //创建QuestionBank类原型，并提供创建与原型相似的目标对象
```

#### 五、单例**模式**
> **定义：** 一个类只有一个实例，且该类能自行创建这个实例的一种模式 
> **意图：减少内存开销；避免对资源的多重占用；优化共享资源**
> **特点：** 没有接口，扩展困难；不利于代码调试；容易违背单一职责原则
