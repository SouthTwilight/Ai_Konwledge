1. JDK、JRE、JVM之间的区别
    
    > ***JDK：*** Java Develpment Kit，是 Java 开发⼯具，Java 开发人员使用 
    > ***JRE：*** Java Runtime Environment，Java 运⾏时环境，Java 程序运行时必需 
    > ***JVM：*** Java Virtual Machine，Java 虚拟机，Java 真正运行 Java 字节码的程序，不同系统环境有不同 JVM
    
2. “ == ” 和 equals ⽅法的区别
    
    > ***“ == ” ：*** 对⽐的是栈中的值，基本数据类型是变量值，引⽤类型是堆中内存对象的地址 
    > ***equals：*** object中默认也是采⽤ == ⽐较，通常会重写为适用于自身的情况
    
3. hashCode 与 equals
    
    >1. 如果两个对象**相等**，则 **hashcode**⼀定也是相同的
    >2. 两个对象**相等**,对两个对象分别调⽤ **equals** ⽅法都返回 true
    >3. 两个对象有**相同 hashcode 值**，它们也**不⼀定相等
    >4. equals 方法被覆盖过，则 hashCode 方法也必须被覆盖**
    >5. hashCode 默认⾏为是 **对堆上的对象**产⽣独特值。如果没有重写 hashCode ，则该类生成的两个对象不会相等，即使它们指向相同数据
    
4. final 关键字及**为什么局部内部类和匿名内部类只能访问外部类的局部 final 变量**
    - final 关键字作用：
        1. 修饰类：表示类不可被继承
        2. 修饰⽅法：**表示⽅法不可被⼦类覆盖，但是可以重载**
        3. 修饰变量：表示变量⼀旦被赋值就不可以更改它的值。
        4. 修饰成员变量：a. **final修饰的是类静态变量**，只能在静态初始化块中指定初始值或者声明该类变量时指定初始值；b. **final 修饰的是成员变量**，可以在⾮静态初始化块、声明该变量或者构造器中执⾏初始值
        5. 修饰局部变量：无默认初始化，局部变量必须由程序员显示初始化（可以创建时初始化或使用前进行赋值初始化）
        6. 修饰基本类型数据和引⽤类型数据：a. 基本数据类型数值⼀旦初始化便不能更改；b.引⽤类型变量初始化后便不能再让其指向另外的对象，但是该对象引⽤的值可变
    - **局部内部类和匿名内部类只能访问外部类的局部 final 变量**
        1. 整个外部类编译之后会生成两/多个 class 文件，一个时主类文件，另外的是在该类中定义的局部/匿名内部类的类文件。
        2. 内部类和外部类是处于同⼀个级别的，内部类不会因为定义在⽅法中就会随着 ⽅法的执⾏完毕就被销毁
        3. 将局部变量设置为 final，对它初始化后保证了内部类的成员变量和⽅法的局部变量的⼀致性
5. String 与 StringBuffer 与 StringBuilder
    > 1. String是不可变的，如果尝试去修改，会新⽣成⼀个字符串对象；StringBuffer和StringBuilder是可变的
    > 2. StringBuffer 是线程安全的，StringBuilder 是线程不安全的，所以在单线程环境下StringBuilder 效率会更⾼；所以尽可能使用 StringBuilder ，当作为多线程的共享变量时使用 StringBuffer 。
    
6. 重载和重写
    > 1. _**重载：**_ 发⽣在**同⼀个类**中，**⽅法名必须相同**，参数类型不同、个数不同、顺序不同，⽅法返回值和访问修饰符可以不同 **(返回值不影响是否重载)** ，编译时发⽣重载。
    > 2. _**重写：**_ 发⽣在父类与子类间，⽅法名、参数列表必须相同，**返回值范围小于等于父类**，**抛出的异常范围小于等于父类**，**访问修饰符范围大于等于父类**；如果⽗类⽅法访问修饰符为 **private** 则**子类不能重写**
1. 接⼝和抽象类
    - 接口与抽象类的区别：
        1. 抽象类可以存在普通成员函数，⽽接⼝中只能存在public abstract ⽅法。
        2. 抽象类中成员变量类型不限，⽽接⼝中成员变量只能是public static final类型的。
        3. 抽象类只能继承⼀个，接⼝可以实现多个
        4. 接口是 like ；抽象类是 is
    - 接口设计目的：
        1. 对类的⾏为进行约束，⼀种“有”的约束，接口规定类需要执行的行为
        2. 接口只约束类的行为有无，但不对如何实现行为进行限制
    - 抽象类设计目的
        1. 进行代码复用，将多个**具有某些相同实现方式行为的不同类**统一派⽣于⼀个抽离出的抽象类；抽象类负责实现这些相同行为，避免子类重复实现，达成代码复用。
8. List 和 Set
    
    > **_List：_** **有序** ，按对象进⼊的**顺序保存**对象，可重复，允许多个Null元素对象，可以使⽤Iterator 遍历元素，使⽤ get(int index) 可进行**随机访问** 
    > ***Set：*** ⽆序，不可重复，最多允许有⼀个Null元素对象，只能⽤ Iterator 接⼝遍历所有元素，不可随机访问
    
9. ArrayList 和 LinkedList
    
    > 1. 底层数据结构，ArrayList 底层基于数组实现，LinkedList 底层基于链表实现
    > 2. ArrayList 更适合随机查找，LinkedList 更适合删除和添加
    > 3. ArrayList 和 LinkedList 都实现了List接⼝；LinkedList 额外实现 Deque 接⼝，故 LinkedList 还可以当做队列来使⽤
    > 4. ArrayList 使用尾插法并指定初始容量可以极大提升其插入添加效率
    
10. HashMap 和 HashTable
    - 两者区别：
        1. HashMap ⽅法无 synchronized 修饰，属于线程不安全类，HashTable 是线程安全类；
        2. **HashMap 允许 key 和 value 为 null，⽽ HashTable 不允许**
    - 底层实现原理
        数组 + 链表实现，直到**链表高度达到 8**、**数组长度超过 64** 时，链表转变为红⿊树，元素以内部类Node节点存在；直到**数据长度低于 6** 则将红⿊树转回数组 + 链表
    
    #📌  **key 为 null，存在下标 0 的位置**

11. ConcurrentHashMap 扩容
    
    > 当某个线程进⾏put时，如果发现 ConcurrentHashMap 正在进行扩容那么该线程⼀起进行扩容；如果某个线程 put 时，发现没有正在进行扩容，则将 key-value 添加到ConcurrentHashMap 中，然后判断是否超过阈值，超过了则进行扩容；ConcurrentHashMap 是⽀持多个线程同时扩容的；先生成一个新数组，在转移元素时，先将原数组分组，将每组分给不同的线程来进行元素的转移，每个线程负责⼀组或多组的元素转移⼯作
