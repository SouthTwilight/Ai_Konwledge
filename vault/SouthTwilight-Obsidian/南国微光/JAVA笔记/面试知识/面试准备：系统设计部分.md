- 系统设计执行步骤：
    1. 明确系统设计要求，包括详细功能要求，以及基本的性能指标
    2. 系统抽象设计：即绘制系统抽象拓扑图
    3. 确定抽象系统需要优化的地方（分布式部署、数据库索引、读写分离等）
    4. 继续完善系统
- 系统构架及常用优化手段：
    1. **高性能构架：数据库读写分离、缓存、负载均衡、异步处理等**
    2. **高可用构架：集群部署、超时重试、降级、限流、熔断、排队等**
    3. **高扩展构架：按需要拆分系统模块**
- 性能测试：
    - 工具：
        1. Jmeter
        2. LoadRunner （付费）
        3. Galtling
        4. Apache Bench
    - 常用软件工具基础 QPS:
        1. Nignx: 单机 30W+ QPS
        2. Redis: 单机 8W+ QPS
        3. Mysql: 单机 4K+ QPS
        4. Tomcat: 单机 2~3W QPS
    - 常遇性能优化问题
        1. SQL 语句是否存在问题
        2. 是否需要升级硬件
        3. 是否需要缓存
        4. 是否存在死锁
        5. 数据库索引使用是否合理
        6. 是否存在内存泄漏
        7. 耗时操作是否异步处理
