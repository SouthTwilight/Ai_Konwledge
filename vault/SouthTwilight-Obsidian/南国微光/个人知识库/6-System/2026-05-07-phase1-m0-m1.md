# AI Knowledge Base V3 — Phase 1 开发计划

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** 搭建个人 AI 知识库核心管道，实现 RSS 文章自动采集 → AI 处理(L1+L2) → Obsidian 存储。

**Architecture:** Hermes Agent (WSL) 编排 + Python 模块化管道 + Obsidian Vault。Extractor 可插拔，统一 Article 数据模型贯穿全管道。

**Tech Stack:** Python 3.11, trafilatura, feedparser, DeepSeek API (L1/L2), Obsidian Markdown + YAML frontmatter

## M1: 核心管道（7 个任务）

### T1.1 实现 RSS Extractor
- 输入: T0.3 Article 模型 + T0.5 config
- 产出: `pipeline/extractors/rss_fetcher.py` — 抓取 RSS + trafilatura 提取正文
- 验证: 对真实 RSS 源（如 Hacker News）成功提取文章列表
- 关闭: commit

### T1.2 实现 Web URL Extractor
- 输入: T0.3 Article 模型
- 产出: `pipeline/extractors/web_extractor.py` — 接收 URL → 提取正文
- 验证: 传入测试 URL 返回干净的 Article 对象
- 关闭: commit

### T1.3 实现去重器
- 输入: T0.3 Article 模型
- 产出: `pipeline/utils/dedup.py` — URL 去重 + 内容哈希去重
- 验证: 重复 URL 被正确过滤
- 关闭: commit

### T1.4 实现 L1 过滤器 (DeepSeek-V3)
- 输入: T0.3 Article 模型
- 产出: `pipeline/processors/l1_filter.py` — 相关性评分(1-10) + 分类标签
- 验证: 对测试文章输出评分和标签
- 关闭: commit

### T1.5 实现 L2 摘要器 (DeepSeek-R1)
- 输入: T0.3 Article 模型
- 产出: `pipeline/processors/l2_summarizer.py` — 结构化摘要 + 关键要点
- 验证: 输出符合 schema 的结构化摘要
- 关闭: commit

### T1.6 实现 Obsidian 格式化器
- 输入: T0.3 Article 模型 + T0.4 vault 目录
- 产出: `pipeline/formatters/obsidian_writer.py` — Markdown + YAML frontmatter + WikiLinks
- 验证: 输出文件可被 Obsidian 正确解析
- 关闭: commit

### T1.7 实现端到端管道
- 输入: T1.1-T1.6 所有模块
- 产出: `pipeline/main.py` — CLI 入口，串联 Extractor → 去重 → L1 → L2 → Obsidian
- 验证: `python main.py --source rss --dry-run` 端到端运行成功
- 关闭: commit
